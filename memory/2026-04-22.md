## 2026-04-22

**Issue: Payment details not showing in order detail after successful payment**

### Root Cause
The Midtrans payment provider (`midtrans_provider.py`) didn't have mappings for all payment types that Midtrans can return. When fetching payment details for orders with payment types like "echannel", "cstore", "bca_va", "mandiri_va", etc., the provider threw an error and returned a 500 status code.

### Fix Applied
Updated the `get_payment_method` function in `/Users/hanggi/projects/ec-mono/ec-payment/src/ec_payment/provider/midtrans_provider.py` to handle more payment types:
- Added mappings for QRIS ("qris", "bca_klikpay") → PaymentMethod.EWALLET
- Added mappings for e-channel and convenience store types ("echannel", "cstore") → PaymentMethod.BANK_TRANSFER
- Added mappings for various bank virtual accounts ("bca_va", "mandiri_va", "bri_va", "bni_va", "permata_va") → PaymentMethod.BANK_TRANSFER
- Made the default fallback return PaymentMethod.BANK_TRANSFER instead of EWALLET

### Steps Taken
1. Identified the issue by checking payment service logs showing "Unknown payment type: echannel" errors
2. Updated the get_payment_method function with comprehensive payment type mappings
3. Restarted the ecom-payment container to apply changes
